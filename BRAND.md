# Vertex — Brand Identity

## Имя

**Vertex** — точка схождения, узел графа, вершина.

Бренд проекта на всех уровнях: бинарники (`vtx-*`), Swift-пакет (`VertexCore`), iOS/macOS приложения, документация, репозиторий. Раньше проект назывался *broker-tunnel* (рабочее техническое имя); историческое имя осталось только в MIGRATION.md как контекст rolling production migration.

## Почему Vertex

Имя работает на нескольких слоях одновременно. Все они, не сговариваясь, описывают одну и ту же архитектурную истину.

### Слой 1 — Теория графов

Vertex (V) — узел графа, в котором сходятся edges (E). Брокер в этой системе буквально является вершиной графа:

```
Client ─edge─→ ⬢ Vertex ←─edge─ Exit
                (broker)
```

Каждый брокер — отдельный vertex. Клиенты и exit-ноды — edges, сходящиеся в нём. Multi-broker fan-out — это многовершинный граф (текущая топология: YC + Sber).

### Слой 2 — Физика частиц (диаграммы Фейнмана)

В квантовой теории поля **vertex** — точка взаимодействия частиц. Частицы встречаются в вершине, обмениваются виртуальными бозонами, расходятся. Каждая вершина имеет константу связи — тип взаимодействия.

Это в точности то, что делает брокер: пакеты "встречаются" в вершине, обмениваются, расходятся к exit/client. Брокер — точка взаимодействия, не хранилище.

### Слой 3 — Этимология

Латинское *vertex / vortex* от *vertere* — "поворачивать". Изначально "макушка головы", "точка вращения". Метафора: точка, вокруг которой всё крутится.

### Слой 4 — Восприятие пользователя

Для непосвящённого — просто красивое технологичное слово. Никаких намёков на VPN, обход цензуры, туннели. Бренд звучит как премиальный продукт сетевой инфраструктуры.

## Как читать имя

Только посвящённый (математик, физик, сетевик) увидит, что Vertex — это технически точное описание брокера как графовой вершины. Для остальных это просто бренд. Это и есть design intent: имя одновременно скрывает суть и описывает её для тех, кто способен увидеть.

## Иконография

Текущее предпочтение:

- **Asterisk-подобный знак** ✱ — лучи, расходящиеся из одной точки. Простой, узнаваемый, напрямую отражает концепт vertex (точка с расходящимися edges)
- **Стилизованная буква V с лучами** — более брендированный вариант: V как корень слова, лучи как edges, исходящие из вершины

Альтернативы для будущего рассмотрения:

- Граф из нескольких точек с подсвеченной вершиной (отсылка к multi-broker fan-out)
- Геометрический конус/треугольник с подсвеченной вершиной
- Чистый знак ✱ или ⁂ с одной активной точкой

### Цветовая палитра

- Глубокий тёмно-фиолетовый + электрический синий (геометрическая премиум-эстетика)
- Чистый монохром: чёрный фон + белая геометрия
- Тёмно-синий с золотым акцентом

## Tagline (черновики)

Не выдают VPN, но отражают концепт vertex:

- "Where paths meet"
- "Точка схождения"
- "Every path crosses one point"
- "All edges meet"

## Бренд-домен

Основной домен бренда — **vertices.ru**. От него рождается всё:

- Bundle ID iOS / macOS: `ru.vertices` (main app), `ru.vertices.tunnel` (NEPacketTunnelProvider)
- App Group: `group.ru.vertices`
- Keychain service / access group: `ru.vertices` / `group.ru.vertices`
- Logger subsystems: `ru.vertices` / `ru.vertices.tunnel`
- Mosquitto domain (production): TBD — будет `vertices.ru` зона с SRV-записями `_vtx-exit._tcp.vertices.ru`
- Web (потенциально): `vertices.ru` лендинг

Reverse-DNS Bundle ID `ru.vertices` (без суффикса) — главное приложение. Будущие компаньон-продукты получают суффиксы (`ru.vertices.cli`, `ru.vertices.dashboard` и т.п.). Tunnel network extension — `ru.vertices.tunnel` как дочерний.

## Migration

Полный план поэтапной миграции с историческим именем `broker-tunnel` / `bt-*` / `com.4few.brokertunnel.*` на `vertex` / `vtx-*` / `ru.vertices.*` — см. `MIGRATION.md`. Кодовая база, бинарники, Docker, iOS/macOS Bundle IDs, DNS-зона `vertices.ru`, Let's Encrypt сертификаты, Mosquitto users и server-side configs — всё мигрировано (Phase 2-4 DONE 2026-04-26). Единственный остаток — wire-protocol HKDF info string `"broker-tunnel-v1"`, который сохраняется ради совместимости.

iOS/macOS Bundle ID меняется один раз (`com.4few.brokertunnel` → `ru.vertices`) с потерей App Store-identity. В нынешнем режиме приватного использования рейтинги/установки не критичны, поэтому бренд получает чистый reverse-DNS под собственный домен.
