Настройки сохранены

Конфигурация записана на диск, сервисы НЕ перезапускались.

Что дальше

  1. Выключите R3S (отключите питание).
  2. Перенесите устройство в целевую сеть.
  3. Подключите кабели: ISP → eth0, ваш роутер → eth1.
  4. Включите питание. Через ~60 секунд VPN поднимется автоматически с новой конфигурацией.

Если на новом месте что-то пойдёт не так — http://192.168.42.1/ снова доступен с устройства за вашим роутером (на новом месте, не сейчас). Откройте, отредактируйте, сохраните.

Для диагностики при подозрениях на ошибку (доступ к R3S по SSH через eth1): sudo journalctl -u vtx-inline-wan -e (PPPoE/DHCP), sudo journalctl -u vtx-inline-router -e (VPN-туннель).

Готово

R3S подключается к VPN. Через минуту проверьте интернет на роутере.

Что дальше

  1. Перезагрузите ваш роутер (или подождите пока он сам обновит DNS-сессии).
  2. На устройстве за роутером откройте https://api.ipify.org — IP должен быть зарубежный (не вашего провайдера).
  3. Откройте https://yandex.ru/ping — должно работать быстро (RU-трафик идёт напрямую).

Если вы отправляли форму по Wi-Fi — пароли передавались по незашифрованному HTTP. Рекомендуем сменить пароль точки доступа на вашем роутере (на случай, если рядом кто-то перехватил Wi-Fi-кадры).

Если переедете на другую сеть/провайдера — этот URL (http://192.168.42.1/) всегда доступен с eth1-стороны для смены настроек.
