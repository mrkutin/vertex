# Vertex marketing site

Public marketing site for Vertex (`https://vertices.ru`). Static, multilingual
(EN / RU), built with Astro 4. No analytics, no third-party scripts, no Google
Fonts CDN — everything ships from this repo.

## Stack

- **Astro 4** (TypeScript strict) — static output, MDX content support
- **Tailwind CSS 3** with design tokens via CSS variables
- **Self-hosted fonts**: Inter Variable + JetBrains Mono (woff2)
- **Icons**: Lucide (inline SVG, no runtime fetch)
- **i18n**: built-in Astro routing, EN at `/`, RU at `/ru/`

## Requirements

- Node 20 or newer (tested on Node 25)
- npm 10+

## Quick start

```bash
cd site
npm install
npm run dev          # http://localhost:4321
```

Other useful scripts:

```bash
npm run build        # static output -> dist/
npm run preview      # serve dist/ locally
npm run check        # astro type-check
```

## Project structure

```
site/
├── astro.config.mjs        # i18n config, integrations, build options
├── tailwind.config.mjs     # design-token bindings to CSS vars
├── tsconfig.json
├── public/
│   ├── fonts/              # self-hosted woff2 (see public/fonts/README.md)
│   ├── icons/              # favicons (svg + raster)
│   └── og-image.png        # social-card placeholder
└── src/
    ├── styles/global.css   # tokens + reset + utilities
    ├── i18n/               # ui dictionary, helpers
    ├── layouts/            # BaseLayout, DocLayout
    ├── components/         # Header, Footer, HeroGraph, Icon, toggles
    └── pages/              # routes (EN at /, RU under /ru/)
```

## Before publishing

1. **Drop the woff2 fonts** into `public/fonts/` — see
   [`public/fonts/README.md`](public/fonts/README.md). The site works without
   them but renders system fallbacks.
2. **Generate raster favicons** from `public/icons/favicon.svg` — see
   [`public/icons/README.md`](public/icons/README.md). Optional in dev.

## Pages

| URL | EN | RU | Source |
|-----|----|----|--------|
| `/`, `/ru/` | ✓ | ✓ | `pages/index.astro`, `pages/ru/index.astro` |
| `/features/`, `/ru/features/` | ✓ | ✓ | `pages/features.astro`, `pages/ru/features.astro` |
| `/download/`, `/ru/download/` | ✓ | ✓ | `pages/download.astro`, `pages/ru/download.astro` |
| `/privacy/`, `/ru/privacy/` | ✓ | ✓ | `content/pages/privacy/{en,ru}.mdx` rendered by `pages/[slug].astro` |
| `/support/`, `/ru/support/` | ✓ | ✓ | `content/pages/support/{en,ru}.mdx` |
| `/terms/`, `/ru/terms/` | ✓ | ✓ | `content/pages/terms/{en,ru}.mdx` |
| `/404.html`, `/ru/404/` | ✓ | ✓ | `pages/404.astro`, `pages/ru/404.astro` |
| `/sitemap-index.xml` | — | — | Generated by `@astrojs/sitemap` |
| `/robots.txt` | — | — | `public/robots.txt` |

To add a new long-form content page (e.g., FAQ), drop two MDX files under
`src/content/pages/<slug>/{en,ru}.mdx` with frontmatter `title`,
`description`, `lastUpdated`. Both routes pick it up automatically.

## Targets

- Lighthouse 100 / 100 / 100 / 100 on Performance / Accessibility / Best
  Practices / SEO.
- `dist/` total weight under 300 KB before adding fonts (fonts add ~500 KB).
- Zero external network requests from any page (verified with DevTools
  Network panel).

## Deploy

Output is static (`dist/`), suitable for Cloudflare Pages, GitHub Pages,
Netlify, or any plain HTTPS host. Cloudflare Pages config will land in a
follow-up task.

## What this site is **not** allowed to grow

- ❌ Google Fonts / any external CDN
- ❌ React / Vue / Svelte runtimes (Astro components only)
- ❌ Analytics scripts (Plausible, GA, Yandex Metrika, …)
- ❌ Cookie banners — site sets no tracking cookies
- ❌ `astro-i18next` or other i18n plugins — use Astro's built-in i18n
